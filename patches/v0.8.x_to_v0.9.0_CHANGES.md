# Upstream KRO Changes: v0.8.3 → v0.9.0

Analysis of upstream KRO changes and their impact on function-kro's upgrade.

Based on: https://github.com/kubernetes-sigs/kro/releases/tag/v0.9.0

---

## Executive Summary

v0.9.0 is a **large release** with significant refactoring across all four packages we vendor. Key themes:

1. **CEL Expression type system overhaul** — Expressions are now structured `*Expression` objects (compiled programs + metadata) instead of raw strings. This is the most pervasive change.
2. **Build context memoization** — New `buildContext` type caches CEL compilation artifacts per-build for performance.
3. **Runtime file split** — `runtime/node.go` was split into `node.go`, `node_resolve.go`, `node_readiness.go`, `node_context.go`.
4. **New features** — Cluster-scoped instances, external collections with selectors, `omit()` CEL function, new CEL libraries (json, maps, lists, quantity).
5. **Parser compiles string templates** — String templates like `"prefix-${expr}"` are now compiled to CEL concatenation at parse time.
6. **Metrics everywhere** — New metrics in cel, runtime, and graph packages.

### Impact Assessment

| Impact | Area | Details |
|--------|------|---------|
| **HIGH** | `graph/builder.go` | Major refactoring: `buildContext`, `RGDConfig`, restructured `buildInstanceNode`, new `Expression` types throughout |
| **HIGH** | `graph/variable/variable.go` | `FieldDescriptor` restructured: `Expressions []string` → `Expression *Expression`, removed `ExpectedType`, `StandaloneExpression`, `Dependencies` |
| **MEDIUM** | `runtime/node.go` | Split into 4 files; `resolve()` unified method replaces separate `GetDesired`/`GetDesiredIdentity` internals; `normalizeNamespaces` moved to `node_resolve.go` |
| **MEDIUM** | `cel/` package | New `Expression` type, `conversion/` package, `sentinels/`, `unstructured/`, many new libraries |
| **MEDIUM** | `graph/parser/` | Returns `*Expression` objects; compiles string templates to concatenation |
| **LOW** | `graph/node.go` | New `NodeTypeExternalCollection`; `NodeMeta` unchanged (still has GVR/Namespaced) |
| **LOW** | `graph/dag/dag.go` | Heap-based topological sort (internal optimization) |
| **LOW** | `metadata/` | Label migration to `internal.kro.run/` prefix; new `selectors.go` |
| **NONE** | `cel/environment.go` | `DefaultEnvironment` API unchanged (internal caching refactor) |

## Detailed Changes by File

### `graph/builder.go` — HIGH IMPACT

#### Signature Changes

| Function | v0.8.3 | v0.9.0 |
|----------|--------|--------|
| `NewBuilder` | `(clientConfig *rest.Config, httpClient *http.Client) (*Builder, error)` | **Same** |
| `NewResourceGraphDefinition` | `(originalCR *v1alpha1.ResourceGraphDefinition) (*Graph, error)` | `(originalCR *v1alpha1.ResourceGraphDefinition, rgdConfig RGDConfig) (*Graph, error)` |
| `buildRGResource` | `(rgResource *v1alpha1.Resource, order int) (*Node, *spec.Schema, error)` | `(p *parser.Parser, rgResource *v1alpha1.Resource, order int, instanceNamespaced bool) (*Node, *spec.Schema, error)` |
| `buildInstanceNode` | (was on Builder, took CRD and schema) | Now a **package-level function** with different params |
| `extractDependencies` | `(env *cel.Env, expr string, nodeNames []string, iterators []string) ([]string, []string, error)` | `(inspector *ast.Inspector, expr *krocel.Expression, iteratorVars []string) (...)` |

#### New Types

```go
// New config parameter for NewResourceGraphDefinition
type RGDConfig struct {
    MaxCollectionSize          int
    MaxCollectionDimensionSize int
}
```

#### Structural Changes

1. **`buildContext`** (new file `build_context.go`) — Per-build memoization of CEL artifacts. Created in `NewResourceGraphDefinition`, passed to all helpers.
2. **`parser.New(schemaCache)`** — Parser is now created per-build with a schema cache.
3. **`ast.NewInspectorWithEnv()`** — Inspector now created with pre-built env.
4. **`validateAndCompileNode(bc, ...)`** — New unified validation+compilation function.
5. **`buildStatusSchema(bc, ...)`** — Restructured, takes buildContext.
6. **`TypedEnvironmentWithProvider()`** — New CEL function replacing `TypedEnvironment()`.
7. **`collectNodeSchemas()`** — New helper to collect schemas for CEL validation.
8. **`validateIdentityFields()`** — New validation for omit() on identity fields.
9. **`Graph.ResourceSchemas`** — New field mapping node ID to OpenAPI schema.
10. **Feature gates** — `features.FeatureGate.Enabled(features.CELOmitFunction)` checks at two points.

#### What This Means for Our Adaptation

Our `buildInstanceNode` replacement needs to be **completely rewritten** for v0.9.0's structure. The new upstream `buildInstanceNode` is a package-level function (not a Builder method) and takes different parameters. Our version's core purpose is the same (build instance from XR schema + status CEL expressions), but:

- `FieldDescriptor` now has `Expression *krocel.Expression` instead of `Expressions []string`
- Dependencies are tracked on `Expression.References` instead of separately
- We'll need to work with the `buildContext` for compilation
- The `parser.ParseSchemalessResource` return type changed (returns `[]variable.FieldDescriptor` with new structure)

Our `getSchemaWithoutStatus` adaptation is simpler — upstream still produces a `*spec.Schema` from the CRD, we still produce one from our XR schema directly. The function shape stays similar.

### `graph/node.go` — MEDIUM IMPACT

- **`NodeMeta` unchanged** — Still has `GVR` and `Namespaced`. Our adaptation to remove them still applies.
- **New `NodeTypeExternalCollection`** added — For selector-based external refs.
- **`ForEachDimension.Expression`** changed from `string` to `*krocel.Expression`.
- **`Node.IncludeWhen`** changed from `[]string` to `[]*krocel.Expression`.
- **`Node.ReadyWhen`** changed from `[]string` to `[]*krocel.Expression`.
- **`Node.Clone()` updated** — Variables copy now clones `Expression` pointer instead of `Expressions` slice + `Dependencies` slice.

### `graph/variable/variable.go` — HIGH IMPACT

Major restructuring:

| v0.8.3 | v0.9.0 |
|--------|--------|
| `FieldDescriptor.Expressions []string` | `FieldDescriptor.Expression *krocel.Expression` |
| `FieldDescriptor.ExpectedType *cel.Type` | Removed (tracked elsewhere) |
| `FieldDescriptor.StandaloneExpression bool` | Removed (templates compiled at parse time) |
| `ResourceField.Dependencies []string` | Removed (tracked on `Expression.References`) |
| `ResourceField.AddDependencies()` | Removed |

This is pervasive — any code that iterates over `fd.Expressions` or accesses `rv.Dependencies` needs updating.

### `graph/validation.go` — LOW-MEDIUM IMPACT

- `validateResourceGraphDefinition` (renamed from `validateResourceGraphDefinitionNamingConventions`) — Now takes `RGDConfig` parameter for collection size limits.
- `validateTemplateConstraints` — Now takes `instanceNamespaced bool` parameter (for cluster-scoped instance support).
- New `validateIdentityFields()` — For omit() validation.
- New `validateCollectionLimits()` — Enforces max collection size/dimensions.

### `graph/parser/*.go` — HIGH IMPACT

- **`Parser` is now a struct** — `ParseResource()` is a method, not standalone function. Must call `parser.New(schemaCache)` first.
- `ParseResource()` return type changed — Returns `[]variable.FieldDescriptor` with single `Expression *krocel.Expression`.
- **`ParseSchemalessResource()` returns 3 values** — `([]FieldDescriptor, []string, error)`. Extra return is plain field paths without expressions.
- `ParseConditionExpressions()` — Now returns `[]*krocel.Expression` instead of `[]string`.
- **String templates compiled at parse time** — `"prefix-${a}-${b}"` compiled to CEL concatenation expression. `Expression.OriginalTemplate` preserves user's original.
- `extractExpressions` now returns `[]exprMatch` with start/end positions (internal change).

### `graph/dag/dag.go` — LOW IMPACT

- Heap-based topological sort (performance optimization).
- API should be backward-compatible.

### `graph/schema/schema.go` — MEDIUM IMPACT

- **`WrapSchemaAsList()` REMOVED** — Moved to `schema.Cache.WrapAsList()`. Any callers must update.
- **New `NamespacelessObjectMetaSchema`** — For cluster-scoped instance CRDs (no `metadata.namespace` field).
- **New `buildNamespacelessObjectMetaSchema()`** — Clones ObjectMetaSchema without namespace property.
- Our `DeepCopySchema` addition is still needed and should still be appended.

### `graph/schema/cache.go` — NEW

- Per-build schema cache for pointer-stable field lookups.
- Used by parser and buildContext.
- Must be included.

### `graph/schema/resolver/metrics.go` — LOW IMPACT

- Schema resolver metrics changes.

### `cel/expression.go` — NEW, HIGH IMPACT

New `Expression` type wrapping CEL expressions:

```go
type Expression struct {
    Original         string          // The CEL expression string
    OriginalTemplate string          // User's template before compilation (for error messages)
    References       []string        // Resource identifiers this expression accesses
    Program          cel.Program     // Compiled CEL program
}
```

Key methods: `NewUncompiled()`, `NewUncompiledSlice()`, `UserExpression()`, `Eval()`.

This type is used throughout builder, parser, variable, and runtime packages.

### `cel/conversion/` — NEW (replaces `conversions.go`)

- `GoNativeType()` moved here from `conversions.go`.
- Used by `Expression.Eval()`.
- Old `conversions.go` removed.

### `cel/environment.go` — MEDIUM IMPACT

- `DefaultEnvironment()` — API unchanged, internal caching with `sync.Once`.
- `TypedEnvironmentWithProvider()` — NEW function, returns `(*cel.Env, *DeclTypeProvider, error)`.
- `TypedEnvironment()` — Still exists as wrapper.
- `baseEnv()` — New internal cached base environment.
- **`CreateDeclTypeProvider()` — REMOVED.** We call this in our builder.go. Must migrate to `TypedEnvironmentWithProvider`.
- **`UntypedEnvironment()` — REMOVED.** Inline as `DefaultEnvironment(WithResourceIDs(ids))`.
- New libraries registered in base: `ext.TwoVarComprehensions()`, `ext.Bindings()`, `k8scellib.Lists()`, `k8scellib.Quantity()`, `library.Maps()`, `library.JSON()`, `library.Lists()`, `library.Omit()`.

### `cel/ast/inspector.go` — MEDIUM IMPACT

- `NewInspectorWithEnv()` — New constructor that takes a pre-built env.
- Improved loop variable scoping.
- `InspectionResult.UsesOmit()` — New method.
- Two-variable comprehension support.

### `cel/library/random.go` — LOW IMPACT

- Added `random.seededInt(min, max, seed)` for deterministic generation.

### `cel/library/{json,lists,maps,omit}.go` — NEW

New CEL libraries. These are included in the default environment via `DefaultEnvironment()`:
- `json.marshal()`, `json.unmarshal()`
- `maps.merge()`
- `lists.setAtIndex()`, `lists.insertAtIndex()`, etc.
- `omit()` (feature-gated)

### `runtime/node.go` — MEDIUM IMPACT (split)

Split into 4 files:
- `node.go` — Node struct, `GetDesired()`, `GetDesiredIdentity()`, unified `resolve()` method
- `node_resolve.go` — `hardResolveSingleResource()`, `hardResolveCollection()`, `softResolve()`, `normalizeNamespaces()`
- `node_readiness.go` — `IsReady()`, `CheckReadiness()`, readiness evaluation
- `node_context.go` — Template variable context building

Key changes:
- `GetDesired()` and `GetDesiredIdentity()` now both delegate to `resolve(mode)`.
- `normalizeNamespaces` is now a method on `*Node` (was free function), returns `error`, and handles cluster-scoped instances.
- `resolve()` handles new `NodeTypeExternalCollection` case.
- Metrics added for node evaluation timing.

**Our adaptation** (removing `normalizeNamespaces`) now targets:
- Lines 248-253 of `node.go` (the call site in `resolve()`)
- `node_resolve.go` (the function definition)

### `runtime/runtime.go` — MEDIUM IMPACT

- `FromGraph` signature changed: `FromGraph(g *graph.Graph, instance *unstructured.Unstructured, rgdConfig graph.RGDConfig) (*Runtime, error)` — Now takes `RGDConfig`.
- Schema-aware CEL value conversion using `Graph.ResourceSchemas`.
- `getOrCreateExpr` now takes `*krocel.Expression` instead of `string`.
- Template variable wiring changed: single `v.Expression` instead of iterating `v.Expressions`.

### `runtime/eval.go` — HIGH IMPACT

- **`buildEnv()` REMOVED entirely** — CEL environments built at graph time now.
- **`evalRawCEL()` REMOVED entirely**.
- `evalExprAny`, `evalBoolExpr`, `evalListExpr` **no longer take `*cel.Env` parameter** — call `expr.EvalCached(ctx)` instead.
- New `filterContext` function (filters context to only referenced keys).
- `toFieldDescriptors` adapted for single `v.Expression` instead of `v.Expressions`.

### `runtime/collection.go` — LOW-MEDIUM IMPACT

- `cartesianProduct` signature changed: added `maxCollectionSize int` parameter, now returns `([]map[string]any, error)`.
- Collection size limit enforcement.
- External collection handling.

### `runtime/state.go` — MEDIUM IMPACT

- `expressionEvaluationState.Expression` changed from `string` to `*krocel.Expression`.
- New `EvalCached` method that handles caching + evaluation via `Expression.Eval()`.
- `InstanceState` and `NodeState` moved to `api/v1alpha1`.

### `runtime/errors.go` — LOW IMPACT

- New sentinel: `var ErrWaitingForReadiness = errors.New("waiting for readiness")`.
- Used by new `CheckReadiness()` replacing `IsReady()`.

### `metadata/labels.go` — LOW IMPACT

- Label key migration to `internal.kro.run/` prefix.
- New selector label helpers.
- Our import path adaptation still applies.

### `metadata/finalizers.go` — LOW IMPACT

- Our import path adaptation still applies.

### `metadata/groupversion.go` — LOW IMPACT

- Our adaptation (remove `GetResourceGraphDefinitionInstanceGVR`) still applies.

### `metadata/selectors.go` — NEW

- New file for label selector utilities (external collections).
- Should be included.
